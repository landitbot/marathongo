# Fusion 960x544 / 960x560 Integration Notes

This note is specific to:

- ONNX: `/workspace/fusion_branch10_960x544.onnx`
- Engine: `/workspace/fusion_branch10_960x544_b1_fp16.engine`
- Custom lib build output:
  `/workspace/DeepStream-Yolo-Seg/nvdsinfer_custom_impl_Yolo_seg/libnvdsinfer_custom_impl_Yolo_seg.so`

## Model facts

The model input is fixed:

```text
images = [1, 3, 544, 960]
```

The model outputs are:

```text
boxes1 / scores1 / mask1 / proto1
boxes2 / scores2 / mask2 / proto2
```

This is not a standard single-head YOLO-Seg output layout. The current
`nvdsparseseg_Yolo.cpp` cannot parse it correctly without modification.

## What works now

What can be validated immediately:

1. TensorRT engine creation from ONNX
2. `nvinfer` engine loading
3. Raw tensor output attachment with `output-tensor-meta=1`
4. Geometry validation for:
   `1280x720 -> 960x540 -> 960x544 -> 960x560`
5. Headless `nvinfer` bring-up with `nvstreammux`

What still needs custom work:

1. Fusion parser for 8 outputs
2. Custom preprocess if you want timestamp bits encoded into the bottom 4 rows
3. Final metadata merge and `nvdsosd` object / mask drawing

## Recommended architecture

Do not try to use one size for everything.

Use:

- `960x544` for inference tensor
- `960x560` for human-readable display

Recommended split:

```text
1280x720
-> decode
-> VIC scale to 960x540 real image content
-> preprocess branch:
   960x540 content -> 960x544 tensor
-> inference
-> display branch:
   960x540 content -> 960x560 canvas
-> OSD clock / text on the bottom strip
```

## Why 4 pixels is not enough for visible time

Bottom 4 pixels are only enough for machine-readable footer data, for example:

- timestamp bit stripe
- frame index bit stripe
- camera id bits

Bottom 4 pixels are not enough for human-readable clock text. If you want
operators to see time, use the `960x560` display canvas.

## Immediate geometry validation commands

### 1. Decode and scale to 960x540

Run this inside the DeepStream container:

```bash
gst-launch-1.0 -e -v \
  v4l2src device=/dev/video0 ! image/jpeg,width=1280,height=720,framerate=30/1 ! \
  nvv4l2decoder mjpeg=1 ! \
  nvvideoconvert compute-hw=2 interpolation-method=4 ! \
  'video/x-raw(memory:NVMM),format=NV12,width=960,height=540' ! \
  fakesink sync=0
```

## nvinfer debug strategy

Start with `network-type=100` and `output-tensor-meta=1`.

This disables native detector / instance-seg parsing and lets you verify that
all 8 output tensors are present before writing the real fusion parser.

Use one of:

- `config_infer_primary_fusion_960x544_onnx_tensormeta.txt`
- `config_infer_primary_fusion_960x544_engine_tensormeta.txt`

Two important DeepStream constraints were verified in practice:

1. `nvinfer` must receive `NvDsBatchMeta`, so `nvstreammux` is mandatory.
2. `nvcompositor` should be followed by `nvvideoconvert` before `nvinfer` to
   stabilize caps negotiation for the `960x544` inference frame.

### Working headless `nvinfer` validation

This command was validated as a working bring-up path for:

- `1280x720 MJPEG`
- VIC scale to `960x540`
- canvas conversion to `960x544`
- `nvstreammux`
- `nvinfer` engine load

```bash
gst-launch-1.0 -e -v \
  nvstreammux name=mux batch-size=1 width=960 height=544 live-source=1 batched-push-timeout=33000 nvbuf-memory-type=0 ! \
  nvinfer config-file-path=/workspace/DeepStream-Yolo-Seg/config_infer_primary_fusion_960x544_engine_tensormeta.txt ! \
  fakesink sync=0 \
  nvcompositor name=comp sink_0::xpos=0 sink_0::ypos=0 sink_0::width=960 sink_0::height=540 ! \
  nvvideoconvert compute-hw=2 interpolation-method=4 ! \
  'video/x-raw(memory:NVMM),format=NV12,width=960,height=544' ! \
  mux.sink_0 \
  v4l2src device=/dev/video0 ! image/jpeg,width=1280,height=720,framerate=30/1 ! \
  nvv4l2decoder mjpeg=1 ! \
  nvvideoconvert compute-hw=2 interpolation-method=4 ! \
  'video/x-raw(memory:NVMM),format=NV12,width=960,height=540' ! \
  comp.sink_0
```

## Engine

For this static ONNX model, `trtexec` must not use dynamic shape arguments.

Working example:

```bash
trtexec \
  --onnx=/workspace/fusion_branch10_960x544.onnx \
  --saveEngine=/workspace/fusion_branch10_960x544_b1_fp16.engine \
  --fp16 \
  --memPoolSize=workspace:2048 \
  --builderOptimizationLevel=5 \
  --skipInference
```

## Next required code work

### Fusion parser

You need a new parser function, for example:

```text
NvDsInferParseFusionSeg
```

It must:

1. Read `boxes1 / scores1 / mask1 / proto1`
2. Read `boxes2 / scores2 / mask2 / proto2`
3. Decode both heads
4. Merge detections into one metadata stream
5. Attach masks in DeepStream format

### Footer-aware preprocess

If you really want timestamp data inside the bottom 4 pixels of the model input,
you need custom preprocess.

That preprocess should:

1. Take `960x540` real content
2. Create a `960x544` tensor
3. Copy the 540 rows of image content
4. Fill the final 4 rows with machine-readable footer data

Do not try to render human-readable time into 4 pixels.
